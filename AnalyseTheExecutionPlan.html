Analyse the Execution Plan

The SQL Execution Plan is used widely in tuning the performance of the queries by monitoring and troubleshooting how the query executed internally. To be able to take benefits of the SQL execution plan, one needs to understand the different information provided by the SQL execution plan, clear at the surface or hidden between the operators. 

 

A very good guide on how best to interpret the Execution Plan, refer to the article by Ahmed Yaseem (https://www.sqlshack.com/how-to-analyze-sql-execution-plan-graphical-components/)

Also see the Microsoft documentation on this topic (https://docs.microsoft.com/en-us/sql/relational-databases/performance/analyze-an-actual-execution-plan?view=sql-server-2017)
