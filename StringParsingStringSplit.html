String Parsing using STRING_SPLIT()

This is a particularly useful table-valued function that splits a string into rows of substrings, based on a specified separator character. It was introduced into MSSQL T-SQL version 2016.

 

The general syntax of the function is STRING_SPLIT ( string , separator ) where string is any character type (for example, nvarchar, varchar, nchar, or char) and separator is a single character expression of any character type (for example, nvarchar(1), varchar(1), nchar(1), or char(1)) that is used as separator for concatenated substrings.

 

Example:

 

SELECT value FROM STRING_SPLIT('The quick brown fox jumps over the lazy dog.', ' ');

 

The result is:

 

value

The

quick

brown

fox

jumps

over

the

lazy

dog.

 

The useful application of this function is to parse a character separated column in a table and associate each value with its corresponding row. This is achieved using the CROSS APPLY join.

 

Example:

 

A table called Product has the following data

 

ProductId

Name

Tags

1

clothing,road,touring,bike

Using the SQL below

 

SELECT ProductId, Name, value

FROM Product

CROSS APPLY STRING_SPLIT(Tags, ',');

Among other features the function can be used to find Rows by List of Values which replaces the need to use anti-pattern searching (i.e. the LIKE ‘%...%’ function).

 

Example:

 

SELECT ProductId, Name, Tags

FROM Product

JOIN STRING_SPLIT('1,2,3',',')

ON value = ProductId;
